# 今後の展望（ToDo）

本プロジェクトの現在のスコープはサンプル／プロトタイプである。
以下は、本番運用に向けて検討・対応が必要な項目を優先度別に整理したものである。

---

## 優先度: 高（本番化に必須）

### AWS へのデプロイ

- フロントエンド: S3 + CloudFront で SPA をホスティング
- Webhook API: Lambda + API Gateway または ECS
- 現在のアーキテクチャ（フロント・バックエンド分離）はそのまま移行可能
- Vercel は開発・ステージング環境として継続利用も可

### データベース連携

- 回答データを DB に永続化（現在は localStorage のみ）
- ユーザー ID と回答を紐付けて保存
- 候補: DynamoDB（サーバーレス構成の場合）、PostgreSQL（ECS 構成の場合）
- localStorage は一時保存として併用し、送信完了時に DB に書き込む構成が現実的

### サーバーからの質問データ取得

- 現在は JSON ファイルに静的定義されている質問データを API 経由で取得する
- 質問の追加・変更をデプロイなしで反映可能にする
- API エンドポイント: `GET /api/questions`

### エラーハンドリングの強化

- LIFF 初期化失敗時のリトライ機構
- ネットワークエラー時のユーザー通知
- API 通信失敗時のリトライ・フォールバック

---

## 優先度: 中（UX 改善・機能拡張）

### テキスト入力型の質問への対応

- 現在は選択肢のみだが、自由入力（テキスト・数値）の質問タイプを追加
- 質問 JSON の `type` に `"text"`, `"number"` 等を追加
- バリデーションルール（必須、文字数制限、正規表現）の定義

### 回答データの分析ダッシュボード

- 管理者向けの回答集計画面
- 質問ごとの回答分布、条件分岐の通過率
- CSV / Excel エクスポート

### 管理画面（質問編集 CMS）

- 非エンジニアでも質問の追加・編集・並べ替えができる GUI
- 条件分岐のビジュアルエディタ
- プレビュー機能

### 回答の「戻る」機能の改善

- 現在は 1 問ずつ戻る仕様だが、任意の質問にジャンプできる UI
- 回答サマリー画面から特定の質問を修正できる導線

### 完了時アクションの拡張

- 回答内容に応じたパーソナライズメッセージ（Flex Message）
- クーポンや特典の配信
- 外部 API / Webhook への回答データ送信

---

## 優先度: 低（将来的な拡張）

### 多言語対応

- 質問テキスト・UI ラベルの i18n
- ユーザーの LINE 言語設定に応じた自動切替

### A/B テスト対応

- 異なる質問フローの出し分け
- フロー別のコンバージョン率計測
- ユーザーセグメントに応じた質問の分岐

### リッチメッセージ対応

- 質問に画像・動画を添付
- カルーセル形式の選択肢
- Flex Message を使ったリッチな Bot メッセージ

### オフライン対応

- Service Worker によるオフラインキャッシュ
- オフライン時も回答を進められ、オンライン復帰時に送信

### アクセシビリティ対応

- キーボード操作対応
- スクリーンリーダー対応（ARIA 属性）
- カラーコントラスト比の確保
