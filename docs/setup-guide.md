# 環境構築手順

## 1. 前提条件

- Node.js 20.19 以上
- npm
- Git
- LINE アカウント（個人の LINE で OK）
- LINE Developers アカウント

---

## 2. LINE Developers の設定

### 2.1 LINE Developers コンソールにログイン

1. https://developers.line.biz/console/ にアクセス
2. LINE アカウントでログイン
3. 初回の場合は開発者情報を登録

### 2.2 プロバイダーの作成

1. コンソールトップの **「作成」** をクリック
2. プロバイダー名を入力（例: `My Provider`）
3. **「作成」** をクリック

> 以降のチャネルは全て **同一プロバイダー内** に作成すること。

### 2.3 Messaging API チャネルの作成

Webhook（Bot からのメッセージ送信）に使用するチャネル。

1. プロバイダー画面で **「新規チャネル作成」** をクリック
2. チャネルの種類: **「Messaging API」** を選択
3. 以下を入力:
   - チャネル名: 任意（例: `Sample ChatBot`）
   - チャネル説明: 任意
   - 大業種 / 小業種: 任意
   - メールアドレス: 自分のメール
4. 利用規約に同意して **「作成」**

#### Channel Secret の取得

1. 作成したチャネルの設定画面を開く
2. **「チャネル基本設定」タブ**
3. **「チャネルシークレット」** の値をコピー（32文字の英数字）

#### Channel Access Token の取得

1. **「Messaging API設定」タブ** をクリック
2. ページ最下部の **「チャネルアクセストークン（長期）」**
3. **「発行」** をクリック
4. 表示されたトークンをコピー（170文字以上の長い文字列）

#### Webhook の有効化

1. 同じ **「Messaging API設定」タブ** 内
2. **「Webhook URL」** にデプロイ後の URL を入力（後で設定）
3. **「Webhookの利用」** を **ON** にする

#### 応答メッセージの無効化

1. **「Messaging API設定」タブ** 内の **「応答メッセージ」** → **「編集」** をクリック
2. LINE Official Account Manager が開く
3. 左メニューの **「応答設定」**
4. **「応答メッセージ」を OFF** にする

> OFF にしないと、ユーザーがメッセージを送るたびに定型の応答メッセージ（「このアカウントでは個別のお問い合わせを〜」）が返ってしまう。

### 2.4 LINE ログインチャネルの作成

LIFF アプリの登録に使用するチャネル。Messaging API チャネルとは別に必要。

1. 同じプロバイダー画面で **「新規チャネル作成」** をクリック
2. チャネルの種類: **「LINEログイン」** を選択
3. 以下を入力:
   - チャネル名: 任意（例: `LIFF Sample`）
   - チャネル説明: 任意
   - アプリタイプ: **「ウェブアプリ」にチェック**
   - メールアドレス: 自分のメール
4. **「作成」**

#### チャネルの公開

1. 作成した LINE ログインチャネルの設定画面を開く
2. 上部の **「開発中」** をクリック
3. **「公開」** に変更

> 「開発中」のままだと `liff.sendMessages()` 等の一部機能が権限エラーになる。

#### LIFF アプリの登録

1. **「LIFF」タブ** をクリック
2. **「追加」** をクリック
3. 以下を設定:

| 項目 | 値 |
|---|---|
| LIFF アプリ名 | 任意（例: `Chat Form`） |
| サイズ | **Full** |
| エンドポイント URL | デプロイ先の URL（ローカル開発時は `https://localhost:5173`） |
| Scope | **`profile`** にチェック |
| ボットリンク機能 | `On (Aggressive)` |

4. **「追加」** をクリック
5. 表示された **LIFF ID**（`1234567890-xxxxxxxx` 形式）をメモ

### 2.5 取得した値のまとめ

最終的に以下の 3 つの値が必要。

| 値 | 取得元 | 用途 |
|---|---|---|
| **LIFF ID** | LINE ログインチャネル → LIFF タブ | フロントエンドの LIFF 初期化 |
| **Channel Secret** | Messaging API チャネル → チャネル基本設定 | Webhook の署名検証 |
| **Channel Access Token** | Messaging API チャネル → Messaging API 設定 | Bot からのメッセージ送信 |

## 3. ローカル環境の構築

### 3.1 リポジトリのクローン

```bash
git clone https://github.com/silmo-yokohama/line-app-sample.git
cd line-app-sample
```

### 3.2 依存パッケージのインストール

```bash
cd src
npm install
```

### 3.3 環境変数の設定

`src/.env.local` を作成し、取得した値を設定する。

```env
VITE_LIFF_ID=取得したLIFF ID
LINE_CHANNEL_SECRET=取得したChannel Secret
LINE_CHANNEL_ACCESS_TOKEN=取得したChannel Access Token
```

### 3.4 開発サーバーの起動

```bash
npm run dev
```

`https://localhost:5173` で HTTPS の dev server が起動する。

> 初回アクセス時にブラウザの証明書警告が表示された場合は「詳細」→「アクセスする」で許可する。

### 3.5 LINE アプリでの動作確認

1. LINE Developers コンソールで LIFF のエンドポイント URL を `https://localhost:5173` に設定
2. スマートフォンの LINE アプリで公式アカウントのトーク画面を開く
3. `https://liff.line.me/{LIFF_ID}` をトーク内に送信してタップ、またはリッチメニューから起動

---

## 4. リッチメニューの設定

1. https://manager.line.biz/ にログイン
2. 対象の公式アカウントを選択
3. 左メニューの **「リッチメニュー」** → **「作成」**
4. 以下を設定:

| 項目 | 値 |
|---|---|
| タイトル | 任意（管理用） |
| 表示期間 | 任意 |
| メニューバーのテキスト | `メニュー` |
| メニューのデフォルト表示 | 表示する |

5. テンプレートを選択（1 枠がシンプル）
6. 画像を作成またはアップロード
7. アクション設定:
   - タイプ: **リンク**
   - URL: `https://liff.line.me/{LIFF_ID}`
   - アクションラベル: `フォームを開く`
8. **「保存」**
