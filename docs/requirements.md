# 要件定義書: LINE チャット風ステップフォーム（LIFF アプリ）

## 1. プロジェクト概要

### 1.1 目的

LINE 公式アカウント上で動作する LIFF（LINE Front-end Framework）アプリとして、チャット風 UI のステップフォームを提供する。ユーザーは Bot との対話形式で質問に回答し、完了時に LINE トークへお礼メッセージが送信される。

### 1.2 ポジショニング

本プロジェクトはサンプル／プロトタイプとして開発する。将来的な本番運用（AWS デプロイ、DB 連携）を見据えつつ、まずは Vercel 上で LINE 内から動作確認できる状態をゴールとする。

### 1.3 対象ユーザー

LINE 公式アカウントを友だち追加済みのエンドユーザー（スマートフォン利用前提）。

## 5. 質問定義データ構造

### 5.1 JSON スキーマ

```json
{
  "questions": [
    {
      "id": "q1",
      "text": "こんにちは！まず、あなたについて教えてください。どの年代ですか？",
      "type": "single",
      "options": [
        { "value": "10s", "label": "10代" },
        { "value": "20s", "label": "20代" },
        { "value": "30s", "label": "30代" },
        { "value": "40s", "label": "40代以上" }
      ],
      "next": {
        "default": "q2"
      }
    },
    {
      "id": "q2",
      "text": "趣味は何ですか？（複数選択OK）",
      "type": "multiple",
      "options": [
        { "value": "sports", "label": "スポーツ" },
        { "value": "music", "label": "音楽" },
        { "value": "travel", "label": "旅行" },
        { "value": "gaming", "label": "ゲーム" },
        { "value": "reading", "label": "読書" }
      ],
      "next": {
        "conditions": [
          {
            "if": { "includes": "sports" },
            "then": "q3a"
          }
        ],
        "default": "q3b"
      }
    }
  ],
  "completion": {
    "message": "ご回答ありがとうございました！",
    "lineMessage": "アンケートにご回答いただきありがとうございました！"
  }
}
```

### 5.2 フィールド定義

| フィールド | 型 | 説明 |
|---|---|---|
| `id` | string | 質問の一意識別子 |
| `text` | string | Bot が表示する質問テキスト |
| `type` | `"single"` \| `"multiple"` | 選択方式 |
| `options` | Array<{value, label}> | 選択肢一覧 |
| `next.default` | string | デフォルトの次の質問 ID |
| `next.conditions` | Array<{if, then}> | 条件分岐ルール |
| `completion.message` | string | 完了画面に表示するメッセージ |
| `completion.lineMessage` | string | LINE トークに送信するメッセージ |

## 7. デザイン仕様

### 7.0 実装方針

UI の実装には Claude Code の **Frontend-Design スキル**（`/frontend-design`）を使用する。
本スキルにより、汎用的な AI 生成 UI ではなく、独自性のある高品質なデザインを実現する。

**Frontend-Design スキルの適用範囲:**
- 全コンポーネントのビジュアルデザイン・レイアウト
- アニメーション・トランジションの実装
- カラーパレットの適用とデザイントークンの定義
- モバイル UX の最適化（タッチターゲット、スクロール挙動等）

以下のデザイン仕様は Frontend-Design スキルへのインプットとして使用する。

### 7.1 カラーパレット

| 用途 | カラー | 説明 |
|---|---|---|
| プライマリ | `#FFB5C5` | パステルピンク |
| プライマリ（濃） | `#FF8FAB` | ボタンホバー・アクティブ時 |
| アクセント | `#FF6B9D` | 重要な CTA |
| 背景 | `#FFF5F7` | ほんのりピンクがかった背景 |
| Bot 吹き出し | `#FFFFFF` | 白背景 + シャドウ |
| ユーザー吹き出し | `#FFD6E0` | 薄いピンク |
| テキスト（メイン） | `#2D2D2D` | ダークグレー |
| テキスト（サブ） | `#888888` | ミディアムグレー |

### 7.2 タイポグラフィ

- フォント: システムフォント（`-apple-system`, `Hiragino Sans` 等）
- Bot メッセージ: 15px / line-height 1.6
- 選択肢ボタン: 15px / font-weight 600
- ヘッダー: 17px / font-weight 700

### 7.3 アニメーション

- メッセージ表示: フェードイン + スライドアップ（300ms ease-out）
- タイピングインジケーター: ドット 3 つのパルスアニメーション
- ボタンタップ: スケール縮小（0.95）+ 背景色変化
- 画面遷移: スムーズスクロール

## 10. リスクと制約

| # | リスク / 制約 | 影響 | 対策 |
|---|---|---|---|
| 1 | LIFF は LINE アプリ内でしかフル機能テストできない | 開発速度が低下する可能性 | 外部ブラウザモードで基本動作確認。LINE 内テストは最終確認時に実施 |
| 2 | `liff.sendMessages()` は LIFF 内でのみ動作 | 外部ブラウザではお礼メッセージが送れない | 外部ブラウザ時はフォールバック表示を実装 |
| 3 | localStorage は端末依存 | 端末変更で回答進捗が消失 | サンプルとして許容。本番化時に DB 連携で解消 |
| 4 | 条件分岐の複雑化 | JSON 定義が肥大化・管理困難になる | 初版は 2〜3 段階の分岐に留める |
| 5 | LIFF SDK の初期化に時間がかかる場合がある | UX に影響 | ローディング画面を表示して体感速度を改善 |
| 6 | Vercel 無料枠の制限 | アクセス増でレート制限される可能性 | サンプル利用の範囲では問題なし |
